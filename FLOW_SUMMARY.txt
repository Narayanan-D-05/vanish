## THE BIG PICTURE

Alice wants to send 100 HBAR to Bob PRIVATELY
(No one should know Alice → Bob connection)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## THE FLOW (7 Steps)

┌─────────────────────────────────────────────────────────────┐
│ STEP 1: Setup                                               │
├─────────────────────────────────────────────────────────────┤
│ Bob: Generates keys, starts Receiver Agent                 │
│ Alice: Gets Bob's view key                                  │
│ Time: One-time setup                                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ STEP 2: Alice Shields (Deposits)                           │
├─────────────────────────────────────────────────────────────┤
│ Alice: "shield 100"                                         │
│   → Generate secret locally                                 │
│   → Create ZK-proof (2 seconds)                            │
│   → Submit to Pool Manager                                  │
│   → Save secret (CRITICAL!)                                │
│ Time: ~2 seconds                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ STEP 3: Pool Manager Batches                               │
├─────────────────────────────────────────────────────────────┤
│ Pool collects proofs from 5+ people:                       │
│   • Alice: 100 HBAR                                        │
│   • Carol: 50 HBAR                                         │
│   • Dave: 75 HBAR                                          │
│   • Eve: 200 HBAR                                          │
│   • Frank: 125 HBAR                                        │
│                                                             │
│ When batch ready (5 proofs OR 30 min):                     │
│   → Random delay 5-15 min (timing privacy)                 │
│   → Verify all proofs                                      │
│   → Submit batch to Pool Contract                          │
│   → Update Merkle tree                                     │
│   → Announce on HCS topic                                  │
│                                                             │
│ Result: Alice is now 1 of 5 (anonymity set)               │
│ Time: 5-30 minutes + random delay                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ STEP 4: Alice Generates Stealth Address                    │
├─────────────────────────────────────────────────────────────┤
│ Alice: "stealth" (using Bob's view key)                    │
│   → ECDH shared secret                                     │
│   → Generate one-time address                              │
│   → Gives Bob: stealth address + ephemeral key            │
│ Time: ~1 second                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ STEP 5: HCS Announcement                                   │
├─────────────────────────────────────────────────────────────┤
│ Pool Manager publishes to public HCS topic:                │
│   {                                                         │
│     "batch_processed": 5 deposits mixed,                   │
│     "stealth_address": "0x4f76...",  ← For Bob            │
│     "ephemeral_key": "0x3436...",                          │
│     "merkle_root": "0x1234..."                            │
│   }                                                         │
│ Time: ~3 seconds (HCS consensus)                           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ STEP 6: Bob Detects Payment                                │
├─────────────────────────────────────────────────────────────┤
│ Bob's Receiver Agent (running in background):              │
│   → Scans HCS topic continuously                           │
│   → Sees stealth announcement                              │
│   → Checks: "Is this mine?"                                │
│      ECDH(my_view_key, ephemeral_key) == address?         │
│   → ✅ YES! This payment is for me!                        │
│   → Decrypt amount: 100 HBAR                               │
│ Time: Real-time detection                                   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ STEP 7: Bob Claims Funds                                   │
├─────────────────────────────────────────────────────────────┤
│ Bob's Receiver Agent automatically:                        │
│   → Generate withdrawal ZK-proof                            │
│   → Create nullifier (prevents double-spend)               │
│   → Submit to Pool Manager                                  │
│   → Pool verifies proof                                    │
│   → Transfer 100 HBAR to Bob's real account                │
│                                                             │
│ Result: Bob receives 100 HBAR                              │
│ Time: ~10 seconds                                          │
└─────────────────────────────────────────────────────────────┘

## PRIVACY ACHIEVED ✅

What observers see on Hedera:
  • Alice deposited to Pool (mixed with 4 others)
  • Bob withdrew from Pool (separate transaction)
  • ❌ NO LINK between Alice and Bob!

Privacy guarantees:
  ✅ Sender anonymity (1 of 5)
  ✅ Receiver anonymity (stealth addresses)
  ✅ Amount privacy (encrypted)
  ✅ Timing privacy (random delays + batching)

Total time: 5-35 minutes
Anonymity set: 5+ participants

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## THE THREE AGENTS

1. 👤 USER AGENT (Alice)
   ├─ Interface: Direct commands OR AI chat
   ├─ Commands: shield, transfer, balance, stealth
   ├─ Start: npm run start:user
   └─ AI Mode: npm run start:user:ai

2. 🏦 POOL MANAGER (Coordinator)
   ├─ Function: Batch proofs, verify, coordinate
   ├─ Batching: 5 proofs OR 30 min (whichever first)
   ├─ Delays: Random 5-15 min
   └─ Start: npm run start:pool

3. 📡 RECEIVER AGENT (Bob)
   ├─ Function: Scan HCS, detect payments, claim funds
   ├─ Mode: Background (autonomous)
   └─ Start: npm run start:receiver

## COMPARISON: PRIVACY vs REGULAR TRANSFER

REGULAR TRANSFER
├─ Command: transfer 0.0.123456 10
├─ Time: 3 seconds
├─ Privacy: ❌ PUBLIC (everyone sees sender/receiver)
└─ Use: Normal payments

PRIVACY TRANSFER
├─ Command: shield 100 → (batching) → withdraw
├─ Time: 5-35 minutes
├─ Privacy: ✅ ZERO-KNOWLEDGE (mathematically proven)
└─ Use: Financial privacy
